# 公開条件注記 ／ 黒塗り方針と恒久公開許可の範囲 ／ リポジトリ共通 ／

本ファイルは、本リポジトリに収録する全実験に共通して適用する公開条件である。各実験フォルダはこの注記を参照する。実験固有の黒塗り記録は、必要に応じて各実験フォルダ側に残す。

## 公開許可の範囲

本リポジトリの実験資料は、AItoAI のオーナー恒久許可のもとで公開する。許可の範囲は次のとおり。

- 秘匿項目のみを黒塗り・抽象化したうえで、各実験のデータを「実験資料」として公開してよい。これは恒久適用される。
- 公開してよいのは、各実験が示す効能 ／ 何ができるか ／ までである。
- システムの内部実装 ／ どう作るか ／ は公開しない。

## 黒塗り・抽象化の方針

各実験資料の作成にあたり、次の項目は転記・公開しない。これらが含まれていた場合は除去または抽象化する。

- 内部の絶対パス、隔離ディレクトリ名。
- システム内部のモジュール名、問い合わせ手段の実体名。これらは「検証可能な永続記憶への問い合わせ」のような抽象度に置き換える。
- 永続層の物理構造 ／ db・audit・index のレイアウト ／ 、記録の採番規約、検証連鎖ファイルの仕様。
- 記録本文中の内部実験ラベル。
- 生成メカニズム系の語。これらは全ファイルから露出ゼロとする。

## 自社検証である旨

本リポジトリの資料は、AItoAI が自社システムの効能を、自社で検証した記録である。第三者による中立評価ではない。この点は各実験フォルダ README の冒頭、およびリポジトリのトップ ／ `README.md` ／ に明記している。

## 実公開の主体

実際のリポジトリ作成・公開 push はオーナーが手動で行う。本リポジトリの資料はいずれも公開前提案であり、AI が自動で公開することはない。
