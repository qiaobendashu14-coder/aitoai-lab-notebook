# AItoAI 実験資料アーカイブ ／ 公開ラボノート ／

> 注記 ／ 公開前提案 ／: 本リポジトリは、私たち AItoAI が自社システムの効能を自社で検証した実験資料を、1 実験 1 フォルダで蓄積していく公開ラボノートである。実際のリポジトリ作成・公開 push はオーナーが手動で行う。AI が自動で公開・作成することはない。本ファイルは、その公開前提案としてのトップ索引である。

## これは何か ／ 誠実明記 ／

ここに集めるのは、私たち AItoAI が設計・運用しているシステム群の効能を、私たち自身で検証した実験資料である。

第三者による中立評価ではない。自社のシステムを自社で検証した記録であることを、リポジトリの入口で正直に明記しておく。私たちが自分の仕組みの効能と限界を正直に開示していく、その実演としてこの場を公開する。

公開してよいと判断しているのは、各実験が示す効能 ／ 何ができるか ／ までである。システムの内部実装 ／ どう作るか ／ は本リポジトリに含めない。

## 公開範囲注記 ／ リポジトリ共通 ／

- 載せるもの: 各実験が観測した効能の記録。確証・仮説・不明を区別し、射程と弱点も併記する。
- 載せないもの: システムの内部モジュール構成、問い合わせ手段の実体、永続層の物理構造、各種内部規約、その他「どう作るか」に当たる一切。
- 公開条件・恒久公開許可の範囲・黒塗り方針は、リポジトリ共通として `LICENSE_NOTE.md` にまとめている。各実験フォルダはこれを参照する。

## 追記運用 ／ 新規実験が増えたとき ／

新しい実験を収録するたびに、次の手順で本リポジトリへ追記する。

1. リポジトリ直下に、その実験のための新しいフォルダを 1 つ作る。1 実験 1 フォルダを原則とする。
2. フォルダ名は snake_case の実験識別子とする ／ 例: `omc_recall` ／ 。日本語・空白・大文字は使わない。
3. そのフォルダ内に、実験フォルダ README ／ `README.md` ／ を置き、冒頭でリポジトリ全体の案内 ／ `../README.md` ／ と公開条件 ／ `../LICENSE_NOTE.md` ／ へクロスリンクする。
4. 本ルート README の「実験一覧 ／ 索引 ／」テーブルに 1 行追記する。実験名・フォルダ名 ／ リンク ／ ・一言要約・状態を埋める。
5. 公開条件・黒塗り方針はリポジトリ共通の `LICENSE_NOTE.md` を継承する。実験固有の黒塗り記録が必要なときは、その実験フォルダ側に残してよい。
