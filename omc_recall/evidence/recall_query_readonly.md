## このファイルは何か ／ 誠実明記 ／

R2・R3 の実行役に渡した「組織記憶の照会手段」が、**読むことしかできない＝書込・改変系を一切持たない**ことを示すための、構造説明と黒塗り版コード片である。第三者評価ではなく、自社検証の対照群設計を誠実に開示するためのものである。

ここで示すのは、その手段の**操作の意味**だけである。手段の中身＝どう永続記憶へ接続するか・どのモジュールを呼ぶか ／ どう作るか ／ は黒塗りする。生のスクリプトをそのまま貼ることはしない。内部モジュール名・API 実体名・内部絶対パス・永続層の物理構造・記録の採番規約・検証連鎖の仕様は、いずれも除去または「（…・黒塗り）」へ置換した。

## 何のための手段か

実行役が永続記憶に対してできるのは、**キーワードに一致する過去の記録を検索して表示する**ことだけである。編集・運用方針を変更・適用する前に、関連する過去の決定が記録されていないかを確認する用途に使う。

実行役に渡したコマンドの形は次のとおりである ／ 内部パスは黒塗り ／ 。

```
（内部パス・黒塗り）recall_query --contains "キーワード"
（内部パス・黒塗り）recall_query --contains "キーワード" --type episodic
（内部パス・黒塗り）recall_query --contains "キーワード" --json
```

オプションの意味は次のとおりである。

- `--contains TEXT` … 記録本文 ／ タイトル＋内容 ／ への部分一致キーワード ／ 必須 ／ 。
- `--type TYPE` … 記録種別フィルタ ／ episodic / semantic / normative ／ 任意 ／ 。
- `--json` … JSON で出力 ／ 既定は読みやすいテキスト ／ 。

## 読むことしかできないことの示し方 ／ 黒塗り版コード片 ／

照会手段の実体のうち、**読み取り専用であること**を確かめられる骨格だけを抜粋し、接続の具体は黒塗りする。

### 1. 永続記憶への接続は黒塗り

```python
# 検証可能な永続記憶への接続設定。
# 接続先・永続層の物理レイアウト・接続するモジュールは全て（内部実装・黒塗り）。
（永続記憶への接続設定・黒塗り）

# 記憶ストアの読み取り照会機能をロードする。
# 内部モジュール名・関数の実体名は（内部モジュール名・黒塗り）。
照会関数 = （読み取り専用の照会関数・黒塗り）
```

### 2. 実行役が呼べる操作は list ／ 一覧照会 ／ のみ

照会手段が永続記憶に対して発行する操作は、次の 1 種類だけである。

```python
res = 照会関数(
    operation="list",          # ← 一覧照会のみ。create / update / delete 等の書込・改変操作は呼ばない。
    record_type=args.rtype,    # 任意の種別フィルタ
    content_contains=args.contains,  # キーワード部分一致
)
```

ここが要点である。この手段が呼ぶのは `operation="list"` ／ 一覧照会 ／ ただ一つで、新規作成・更新・削除・状態変更といった**書込・改変系の操作を一切持たない**。実行役は永続記憶を読むことだけができ、汚染・改変はできない。

### 3. 結果の表示と「記録なし」の正直な返し

```python
print(f"[query] contains={args.contains!r} type={args.rtype}")
print(f"[result] status={res.get('status')} hits={res.get('count')}")
for rec in res.get("records", []):
    print("record_id :", rec.get("id"))     # ← 記録ID。トランスクリプトでは（記録ID・黒塗り）
    print("type      :", rec.get("type"))
    print("timestamp :", rec.get("timestamp"))
    print("status    :", rec.get("status"))
    print("content   :", rec.get("content"))
if not res.get("records"):
    print("（該当する過去の記録は見つかりませんでした）")
```

ヒットが無いときは「該当する過去の記録は見つかりませんでした」と正直に返す。R2・R3 の実行役が、記録に無いテーマ ／ date notation 等 ／ に対して `hits=0` を受け取り、「記録なし」と正直に報告できたのはこの挙動による。ありもしない決定を作らないという健全性は、ここで担保される。

## この構造が証拠として示すこと

- 実行役に渡したのは**読み取り専用の問い合わせ手段**であり、永続記憶への書込・改変経路を持たない。
- ゆえに、R2・R3 で実行役が復元した過去決定 ／ D-old ／ は、実行役が書き込んだものではあり得ず、**事前に永続記憶へ置かれていた記録を読み出した結果**である。
- 「ある／ないを正直に返す」挙動は、照会結果をそのまま表示し 0 件を 0 件と返す構造に由来する。

すなわち、検証可能な永続記憶への読み取り専用の問い合わせ手段が**存在し**、それが**読むことだけを許し**、**ある／ないを正直に返す**。ここまでは示唆する。その手段をどう配線するか・永続記憶をどう構成するかという実装方法は、本パックでは開示しない ／ 効能まで・実装は非公開 ／ 。
